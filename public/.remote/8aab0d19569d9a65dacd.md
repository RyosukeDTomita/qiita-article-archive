## これは何?

Kali Linuxの環境が欲しくなったのでVMで立てた。
ハマったポイントが結構あったので、書いておく。

### 環境

- PC: [LIFEBOOK WU5/J3](https://www.fmworld.net/fmv/uh_z/)
- ホストマシン: Ubuntu 24.04.3 LTS
- VirtualBox 7.2.6 for Linux
- kali-linux-2025.4-virtualbox-amd64

---


## うまく行った方法

### Virtual Boxインストール

[Virtual Boxのダウンロードページ](https://www.virtualbox.org/wiki/Linux_Downloads)からUbuntu 24.04を選択してダウンロード。

```shell
sudo apt install ./virtualbox-7.2_7.2.6-172322\~Ubuntu\~noble_amd64.deb
```

### Kali Linuxのイメージ取得

https://www.kali.org/get-kali/#kali-virtual-machines

### Secure Boot停止

そのままではVirtual Boxが起動できなかったのでSecure Bootを停止した。

### 起動

- 取得した`.7z`ファイルを展開。
- Virtual Box起動
- Openを押下し、Virtual machine filesを選択する

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/3718390/d9dd7425-4608-43c9-97bd-863f555e6387.png)

:::note info
一度仮想マシンを再起動すると画面サイズにあったサイズでGUIが開けるようになった。
:::

### 日本語化

自分は日本語配列のキーボードを使っているためとりあえず日本語配列のキーボードに変更した。

```shell
setxkbmap jp
```

こちらに日本語化手順を記載

https://qiita.com/sigma_devsecops/items/2ca565568955a8d1e0c6

## VMWareだとうまくいかなかった

https://blog.jbs.co.jp/entry/2025/02/26/130309

:::note info
ダウンロードサイトまで行くのめちゃくちゃわかりにくいので上記のブログを参照。
:::

上記のサイトを参考にしてVMware Workstation Proを入れたが、`Could not open /dev/vmmon: No such file or directory. Please make sure that the kernel module vmmon' is loaded.`が表示されてKali Linuxが起動できなかった。

### 試したこと1 VMware Moduleをビルド

```shell
sudo apt install build-essential dkms linux-headers-$(uname -r)
sudo vmware-modconfig --console --install-all # ここで失敗する
[AppLoader] Use shipped Linux kernel AIO access library. 
An up-to-date "libaio" or "libaio1" package from your system is preferred. 
[AppLoader] GLib does not have GSettings support. 
Stopping VMware services: VMware Authentication Daemon done Virtual machine monitor done Starting VMware services: Virtual machine monitor failed Virtual machine communication interface done VM communication interface socket family done Virtual ethernet failed VMware Authentication Daemon done Unable to start services
```

### 試したこと2 vmware-host-modulesを使って自前でビルドしてみる

https://github.com/mkubecek/vmware-host-modules

:::note info
現在ダウンロードできるVMware WorkStations Proで上記リポジトリでサポートされているのは17.5.2のみである(2026年2月現在)である。
:::

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/3718390/613bdeaf-d982-4a13-ae06-e5983943a060.png)

だが、VMware 17.5.2にした途端、VMWareが起動できなくなり、[Reddit](https://www.reddit.com/r/Ubuntu/comments/1dctsko/vmware_workstation_1752_kernel_module_updater/?tl=ja)でも同様の情報を見つけたため、VMWareにこだわる理由もないのでVirtual Boxで試すことにした。
