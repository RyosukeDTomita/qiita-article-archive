## 発生した事象

1. [Burp Suite](https://portswigger.net/burp)をアップデート
2. 内蔵ブラウザのアイコンをクリック --> ブラウザが起動しない
    ![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/3718390/def73c7f-acc9-4596-a7a8-e59ecfc44e71.png)

### 環境

- Ubuntu 24.04.4 LTS (Noble Numbat)
- Burp Suite Community Edtion v2026.2.3

---

## 修正方法

```shell
cd ~/BurpSuiteCommunity/burpbrowser/145.0.7632.45
sudo chown root:root chrome-sandbox
sudo chmod 4755 chrome-sandbox
```

### おまけ: デバックの流れ

1. Help --> Health check for Burp's browser
    ![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/3718390/e8ac2dff-b5c7-4214-b842-c5e6a8b32c85.png)
2. Burp's browserが起動していないことを確認
    ```
    Browser:
    Live:   N/A
    Total:  0
    ```
3. 直接起動できるかチェック --> 起動できず `make sure that /home/sigma/BurpSuiteCommunity/burpbrowser/145.0.7632.45/chrome-sandbox is owned by root and has mode 4755.`
    ```shell
    cd ~/BurpSuiteCommunity/burpbrowser/145.0.7632.45         1
    ./chrome
    [1053363:1053363:0314/105808.015478:FATAL:sandbox/linux/suid/client/setuid_sandbox_host.cc:166] The SUID sandbox helper binary was found, but is not configured correctly. Rather than run without sandboxing I'm aborting now. You need to make sure that /home/sigma/BurpSuiteCommunity/burpbrowser/145.0.7632.45/chrome-sandbox is owned by root and has mode 4755.
   [1]    1053363 trace trap (core dumped)  ./chrome
   ```
4. 上記の[修正方法](#修正方法)に記載のある通り、権限を付与したら起動できた。
