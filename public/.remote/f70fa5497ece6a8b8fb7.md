## これは何?

https://obsidian.md/changelog/2026-02-27-desktop-v1.12.4/

Obsidian 1.12から[Obsidian CLI](https://help.obsidian.md/cli)が使えるようになった。

Obsidian CLIがあることでノートの管理をLLMに任せるのがやりやすくなりそうなのでいろいろ試してみる。

---

## 環境

```shell
cat /etc/os-release| head -n 1
PRETTY_NAME="Ubuntu 24.04.4 LTS"
obsidian version
1.12.4 (installer 1.12.4)
```

Obsidianが1.12以下の方はアップデートが必要

## とりあえず、Obsidian CLIを動かしてみる

### Obsidian CLIを有効化する。

ObsidianのデスクトップアプリからCLIを有効にする。General --> Advanced --> Command line interface

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/3718390/fabc70e1-3141-4375-a151-0f9dde2391a9.png)

有効にすると`~/.local/bin/obsidian`が作成されPATHが通る。

```shell
which obsidian
/home/sigma/.local/bin/obsidian
```

### CLI経由でdaily noteを編集してみる

:::note info
自分の環境では、
1. Obsidianのデスクトップアプリが起動していない状態だとterminalに`obsidian`と入力した際にObsidian CLIではなく、デスクトップアプリが起動した。
2. Obsidianデスクトップ起動後にterminalに`obsidian`と入力するとCLIが起動した。
現状、Obsidian CLIを使う際にはデスクトップアプリを起動しておく必要がありそう。

![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/3718390/41f06b51-87eb-4860-a9a2-da1e025b2c63.png)

:::

```shell
obsidian daily:append content="- [ ] test"
```
Daily ノートの末尾にタスクを追加できた。
![image.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/3718390/b36eb942-42d5-4c01-a51f-b4fb9f5f2ed1.png)

## Skillsに使うファイル検索系のコマンド

```shell
obsidian search query=TDD path=Clippings limit=3 format=text
Clippings/論理的思考力と論理的検証力は違うかもという話.md
Clippings/自動テストとテスト駆動開発、その全体像.md
Clippings/低スキル志向な現場とは何か.md

obsidian search:context query=Haskell
Clippings/Algebraic data type.md:76: - [Concrete data type](https://wiki.haskell.org/index.php?title=Concrete_data_type "Concrete data type")
Clippings/Algebraic data type.md:77: - [Concrete view](https://wiki.haskell.org/index.php?title=Concrete_view "Concrete view")
Clippings/Algebraic data type.md:78: - [Indirect composite](https://wiki.haskell.org/index.php?title=Indirect_composite "Indirect composite")
```

[公式ドキュメント](https://help.obsidian.md/cli#%60search%60)に明記されていないですが、tag検索とかも効きそう。

```shell
obsidian search query=tag:関数型言語 format=text | wc -l
18 # tag検索
obsidian search query=関数型言語 format=text | wc -l
29 # 文字列検索
```

曖昧検索用の[Skills](https://support.claude.com/en/articles/12512176-what-are-skills)を作ってみた。

1. ユーザがファイルの探索を依頼

### ファイル一覧表示してrename

一覧表示してタイポしているものやファイル名がイケてないものをrenameしてもらうとか良さそう。



### (テンプレートからファイル作成)

これはAI関連なくやりたいこと。

自分は技術書のメモを取る際には内容ごとにObsidianのノートを分割している。
本に対応するtemplateを指定してShell Scriptでノートを作るとかに良さそう。

```shell
obsidian create template=effective_java path=Literature name=Static Factoryメソッド open
```
