## 原因

```
git rebase -i HEAD~30
```

のようなコマンドを実行した際に数をでかくしすぎて関係のないコミットが混ぜてしまった。

---

## 解決策

事前に現在のファイルをバックアップ

```shell
git reflog
git reset --hard HEAD@{<n>} # rebase前に戻る
退避したファイルをもとにリベース前と現在のファイル差分をコミットする
```

(普通にrebase自体を取りやめるでもいいと思うが、自分の場合は上にコミット積んでいたのでだるかった)
